# Основная информация

Программа демонстрирует работу с пользовательскими аннотациями и их обработчиками, включая кэширование, валидацию, автоматический вызов методов и генерацию строкового представления объектов. Реализация использует Reflection API для анализа и выполнения действий в зависимости от применённых аннотаций.

# Основные классы и методы
Все методы и классы расположены в папке src:

<h2> 1) Раздел main/java/org/example </h2>

В данном разделе расположен файл Main.java и пакет ru.Dolgorukov

<h3> Main.java </h3>
Основной файл main. В данном файле представлена демострация работы методов и аннатаций.

<h3> пакет ru.Dolgorukov </h3>

В данном пакете находится 3 пакета: additionalclasses, annotation, annotationhandlers.

<h2>- additionalclasses </h2>

В данном пакете находится класс TestAnnotaion, к классу применены различный аннотации для демострации их работы. 

Аннотации класса:
- @Cache({"string1","string3"})
- @Two(first = "First",second = 2)
- @Validate({String.class, Integer.class})
- @ToString
- @Default(String.class)

Информация о аннотациях буде преставлена далее.

Поля класса:
- String string1 - применена аннотация @ToString(ToString.Mode.NO)
- String string2 - простое поле
- String string3 - простое поле

Методы в классе:
- method1 - выводит строку "метод 1" (Применена аннотация @Invoke)
- method2 - выводит строку "метод 2"
- method3 - выводит строку "метод 3"

<h2>- annotation </h2>

В данном пакете представлены созданные аннотации:
- @Cache - Характеристики: Целью может быть ТИП. Доступна во время исполнения программы. Имеет необязательное свойство value, типа String[]. Значение свойства по умолчанию: пустой массив.
- @Default - Характеристики: Целью может быть ТИП или ПОЛЕ. Доступна во время исполнения программы. Имеет обязательное свойство value типа Class.
- @Invoke - Характеристики: Целью может быть только МЕТОД. Доступна во время исполнения программы. Не имеет свойств.
- @ToString - Характеристики: Целью может быть ТИП или ПОЛЕ. Доступна во время исполнения программы. Имеет необязательное свойство valuec двумя вариантами значений: YES или NO. Значение свойства по умолчанию: YES.
- @Two - Характеристики: Целью может быть ТИП. Доступна во время исполнения программы. Имеет два обязательных свойства: first типа String и second типа int.
- @Validate - Характеристики: Целью может быть ТИП или АННОТАЦИЯ. Доступна во время исполнения программы. Имеет обязательное свойство value, типа Class[].

<h2>- annotationhandlers </h2>

В данном пакете представлены обработчики для аннотаций:
- CacheHandler - Выводит список всех кэшируемых областей (все параметры, переданный в аннотацию @Cache) или сообщение, что
список пуст
- DefaultHandler - Выводит имя указанного класса по умолчанию (параметр аннотации Default)
- InvokeHandler - Через Reflection API находит методы, отмеченные аннотацией @Invoke, и вызывает их автоматически
- ToStringHandler -  Формирует строковое представление объекта, учитывая только те поля,
где @ToString имеет значение YES
- TwoHandler - Считывает и выводит значения свойств, переданных в аннотации @Two
- ValidateHandler - Выводит классы указанные в аннотации

<h2> 2) Раздел test/java </h2>

Содержит тесты (Используется фреймворк JUnit):
- InvokeTest - Проверяет корректность вызова методов,отмеченных аннотацией @Invoke.
- TwoTest - Проверяет корректность обработки аннотации @Two, если её свойства заданы некорректно. Например, строковое свойство first пустое(""), а числовое second отрицательное.
